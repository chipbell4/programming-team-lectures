Setting Up To Work

Judges test your code like this:

  • They download the code you uploaded
  • They compile it (if applicable): 
    javac MySolution.java
  • They run your program, using a judging data set: 
    java MySolution < judgingInput.txt > yourOutput.txt
  • They then check that your output matches the correct output: 
    diff yourOutput.txt judgingOutput.txt

Your Workflow should therefore be similar

  • Write your code in an editor
  • Make sure it compiles
  • Run your program with a testing dataset: 
    python MySolution.py < testInput
  • Submit it to the judges

Writing code

  • In Linux, I use Vim.
  • In Windows, Notepad++ and Eclipse do fine as well
  • Regardless, learn keyboard shortcuts to speed up your workflow: VIM, Notepad++, and Eclipse have tons of shortcuts.
  • Vim folks, here's some exercises

Compiling Your Code

  • If you're coding in Python, nice work! You're already done
  • In Vim, type :!javac % to compile the current file you're editing. You can run any shell command this way
  • Eclipse builds automatically as you code. However, Ctrl + B builds manually if you need to
  • Notepad++ is just an editor. You'll need to open cmd.exe and run it manually. cd to where your code is, and then run c:\program files\java\jdk1.6.0_16\bin\javac.exe yourfile.java. You'll probably need to tweak that directory to match where java's located, but that should get you started

Running Your Code

  • Typing input into your program by hand is frustrating and error-prone. Use file redirection if possible!
  • Regardless of OS, the basic formula is this:
    • cd into the directory where your code lives
    • Run java MyClass < inputfile.txt (or python myfile.py < inputfile.txt
    • You may have to use the full path to the runtime if it's not in your path (generally a Windows issue)
    • For Java: c:\program files\java\jdk1.6.0_16\bin\java.exe should get you started
    • For Python: c:\Python33\python.exe might work

Debugging Problems

Types of Incorrect Solutions

  • Compile error - Your code didn't compile
  • Presentation Error - Your output looked right, but there were some formatting issues
  • Runtime Error - An error occurred while your program was running
  • Time Limit Exceeded - Your program took too long to finish
  • Wrong Answer - You got the wrong answer...

Mistakes You Shouldn't Make

  • Your code should always compile.
  • Your code should never have presentation errors.
  • These mistakes are generally caused by not paying attention, and not real programming error

Runtime Errors

  • Generally caused by array indexing issues (but not always).
  • Look for where you might be indexing an array and see if there's a way it could go too high
  • Try harder cases and see if you can replicate the issue

Time Limit Exceeded

  • This generally means your algorithm is inefficient. Rethink it a little.
  • It could also means there's an infinite loop somewhere. Make sure there's not weird case like that
  • Try harder cases and see if you can replicate the issue.

Wrong Answer

  • It's not very descriptive, but it looks like your algorithm is incorrect.
  • Try different cases, especially ones you know the answer to and try to replicate the issue
  • This is the most frustrating part. Really think about the problem. Think about edge cases, large cases, small cases, everything!

Exercise (Edge Cases)

In the previous problem, I didn't specify limits on the numbers. In general, a problem will specify limits and those limits are very important. For instance, let's say for the first problem the positive integer could be up to 100. Can your problem handle that case? What if the number were 2147483647? What will your solution say? What should it say? What about 2147483648?

Fix your code to pass those cases.
